TRADE PLANNER WARNING LOGIC UPDATE

The Trade Planner was built before Buy Limit orders existed.

Now that Buy Limits are live, the current warning message can be misleading.

CURRENT BEHAVIOR

When:

Entry MC < Current Market Cap

the planner displays:

“Entry MC is below current market cap. Results represent a hypothetical earlier entry.”

This was correct before Buy Limits existed, but now there are two different scenarios:

SCENARIO 1 — MANUAL / HYPOTHETICAL ANALYSIS

Buy Limit is NOT enabled.

Example:

Current MC = 250K
Entry MC = 150K

In this case the warning is correct because the user is analyzing a past entry.

Display:

“Earlier Entry Scenario. Calculations assume you entered at 150K MC rather than the current market cap.”

SCENARIO 2 — BUY LIMIT ENABLED

Buy Limit IS enabled.

Example:

Current MC = 250K
Entry MC = 150K
Buy Limit enabled

In this case the user is NOT analyzing a past entry.

The user intends to place a future Buy Limit order.

Display:

“Buy Limit Scenario. Order will remain pending until market cap reaches 150K MC.”

or

“This trade plan is based on a Buy Limit order filling at 150K MC.”

GOAL

Update warning logic so it reflects the actual workflow.

RULES

IF:

* Entry MC < Current MC
* Buy Limit disabled

Show:
Earlier Entry Scenario warning.

IF:

* Entry MC < Current MC
* Buy Limit enabled

Show:
Buy Limit Scenario warning.

IF:

* Entry MC >= Current MC

Show no warning.

VISUALS

Keep warning styling.

Use:

* amber/yellow for Earlier Entry Scenario
* blue/info styling for Buy Limit Scenario if easy
* otherwise reuse existing warning style

DO NOT CHANGE

* Risk/reward calculations
* Profit calculations
* Buy Limit logic
* TP/SL logic
* Order execution
* Portfolio logic
* Leaderboard logic
* Database schema

This is a messaging and UX clarification update only.

QA

Test:

1. Entry below current MC + Buy Limit OFF
    → Earlier Entry warning appears.
2. Entry below current MC + Buy Limit ON
    → Buy Limit warning appears.
3. Entry equal to current MC
    → No warning.
4. Entry above current MC
    → No warning.
5. Typecheck passes.

Return:

* files changed
* warning logic changes
* screenshots if possible
* confirmation calculations remain unchanged
* confirmation safe to publish
