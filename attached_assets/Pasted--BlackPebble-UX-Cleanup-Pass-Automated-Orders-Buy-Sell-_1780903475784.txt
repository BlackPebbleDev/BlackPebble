# BlackPebble UX Cleanup Pass — Automated Orders + Buy/Sell Experience Optimization

IMPORTANT:
Do NOT modify any trading logic, order execution logic, database schema, API endpoints, calculations, slippage engine, TP/SL engine, Buy Limit engine, portfolio math, or leaderboard functionality.

This task is purely a usability and interface improvement focused on making BlackPebble easier for new traders while keeping all existing functionality intact.

====================================================
1. RENAME "EXIT ORDERS" TO "AUTOMATED ORDERS"
====================================================

Current:

Exit Orders
- Take Profit
- Stop Loss

Replace with:

Automated Orders

Reason:
BlackPebble now supports Buy Limits, Take Profit, and Stop Loss. "Exit Orders" is no longer accurate because Buy Limits are entry orders.

====================================================
2. MOVE BUY LIMIT INTO AUTOMATED ORDERS
====================================================

Currently Buy Limit is separated from TP/SL.

Move Buy Limit into the same Automated Orders section.

New structure:

▼ Automated Orders

☐ Buy Limit
Buy when MC ≤ [value]

☐ Take Profit
Sell when MC ≥ [value]

☐ Stop Loss
Sell when MC ≤ [value]

Maintain all existing functionality.

No backend changes.
No order logic changes.

This is only a UI reorganization.

====================================================
3. ADD BEGINNER HELP TEXT
====================================================

Under Automated Orders add:

"Automated orders execute automatically when your conditions are met.

Buy Limit → enters a position
Take Profit → locks in gains
Stop Loss → limits losses"

Keep subtle and mobile friendly.

====================================================
4. IMPROVE ORDER LABELS
====================================================

Current:

Target MC
Stop MC

Replace where appropriate with:

Take Profit MC
Stop Loss MC

Goal:
Make terminology obvious to new traders.

====================================================
5. SIMPLIFY THE BUY / SELL BOX
====================================================

The Buy/Sell area currently feels cluttered.

Prioritize:

- Amount
- USD/SOL toggle
- Quick amount buttons
- Buy/Sell button

These should be the most visually important items.

====================================================
6. COLLAPSE TRADE DETAILS BY DEFAULT
====================================================

Current trade details:

- Execution Price
- Estimated Receive
- Estimated Slippage
- Trade Impact

Move these into a collapsible section.

Example:

▼ Trade Details

Expanded shows:

Execution Price
Estimated Receive
Estimated Slippage
Trade Impact

Collapsed by default.

Reason:
New traders should not be overwhelmed by advanced execution data.

Market Cap

====================================================
8. IMPROVE PAGE FLOW
====================================================

Reorder Trading page sections so they appear in this order:

1. Token Information
2. Buy / Sell Box
3. Automated Orders
4. Trade Planner
5. Active Orders
6. Positions / History

Goal:

What am I trading?
→ How much do I want to buy?
→ Do I want automation?
→ Do I want advanced planning?

This is more intuitive for beginners.
