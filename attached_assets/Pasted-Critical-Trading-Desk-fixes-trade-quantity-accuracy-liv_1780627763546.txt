1. FIX PAPER TRADE QUANTITY CALCULATION
====================================================

Problem:
The app showed “Bought PEG — 49.00 SOL → 637.5K tokens” for a very low market cap token around $70K-$80K MC. That token quantity appears incorrect.

Audit and fix buy/sell token quantity calculations.

Requirements:

- Use actual token price, not market cap, FDV, chart value, or displayed formatted value.
- Use DexScreener pair.priceUsd or trusted hydrated token price for tokenPriceUsd.

Correct formula:

amountInUSD = solAmount * currentSolUsdPrice
tokenQuantity = amountInUSD / tokenPriceUsd

Never use:
- marketCap
- fdv
- formatted market cap text
- chart display value
- rounded UI text

for trade execution calculations.

Store full precision internally.
Only format for display.

If tokenPriceUsd is missing, zero, stale, or invalid, block trade and show:

“Price data unavailable. Trade not executed.”

Add development-only debug logging:
- SOL amount spent
- SOL/USD price used
- token price used
- calculated USD amount
- calculated token quantity
- token mint
- DexScreener pair used

Verify:
- Buy quantity
- Position size

- Portfolio equity

This is confusing.

Use this panel for BlackPebble paper trade activity first.

“Recent Paper Trades”

“Live Paper Trades”

3. FIX LIVE STATUS HONESTY
====================================================

Do not show green LIVE unless data is actually updating.

Status rules:

Green LIVE:
- Feed connected
- Events updating successfully

Yellow DELAYED:
- No events in more than 2 minutes
- Feed stale

Gray IDLE:
- No selected token
- No trade history yet
- Paper trading ready but no trades made

Red OFFLINE:
- API/WebSocket error
