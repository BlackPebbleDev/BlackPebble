BLACKPEBBLE ADMIN CONTROL CENTER + STANDALONE THESIS INTEGRATION

Goal:
Update BlackPebble so admins can safely manage/test the new social features, and separate token theses from token calls.

Current issue:
A user can write a thesis only while calling a token. That is too limiting.

New model:
- Call Token = public prediction + optional thesis
- Thesis = standalone research/analysis post without creating a tracked call

Important:
Do NOT remove the optional thesis field from Call Token.
Call Token should still allow an optional thesis.

Do NOT change trading calculations.
Do NOT change leverage math.
Do NOT change portfolio accounting.
Do NOT change leaderboard formulas except where admin reset/cleanup requires recalculation.
Do NOT change the BlackPebble V2 design language.

====================================================
PART 3 — STANDALONE THESIS FLOW
====================================================

Add a separate Thesis button on token pages.

Purpose:
Allow users to publish research, thoughts, or analysis about a token without creating a tracked call.

Thesis is NOT a price prediction unless the user also chooses to call the token.

Standalone Thesis should NOT affect:
- Top Caller ranking
- Call multiple
- Call hit rate
- Call history

Standalone Thesis SHOULD affect:
- Thesis history
- Token thesis list
- Feed activity
- Profile thesis section

Only X-authenticated users may publish a thesis.

If a guest or wallet-only user clicks Thesis, show:

"Connect X to publish token theses and build your BlackPebble profile."
