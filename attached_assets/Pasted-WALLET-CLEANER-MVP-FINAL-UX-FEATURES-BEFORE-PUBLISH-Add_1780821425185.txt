WALLET CLEANER MVP — FINAL UX FEATURES BEFORE PUBLISH

Add these features only to the Wallet Cleaner utility.

Do NOT modify:
- paper trading
- portfolio
- leaderboard
- X login
- wallet login
- market feeds
- charts
- API server
- database

Only modify wallet-cleaner related files.

====================================================
1. MULTI-ACCOUNT BATCH UX
====================================================

The Wallet Cleaner should clearly support selecting and closing multiple empty token accounts.

Requirements:
- Users can select multiple closeable accounts with checkboxes.
- Add Select All and Deselect All controls.
- Nothing should be auto-selected on initial scan.
- Selected accounts should be batched into transactions.
- Use max 10 close instructions per transaction to avoid Solana transaction size issues.
- Show how many transactions will be required before signing.

Example:
27 accounts selected
3 transactions required

====================================================
2. RECOVERY SUMMARY CARD
====================================================

Before signing, show a clear summary:

- Accounts selected
- Recoverable SOL
- Estimated network fees
- Estimated net received
- Number of transactions required

Example:

Accounts selected: 27
Recoverable SOL: 0.0551
Estimated network fees: 0.000015
Estimated net received: 0.055085
Transactions required: 3

Recoverable SOL must be calculated from actual token account lamports, not hardcoded.

====================================================
3. PREVIEW MODAL
====================================================

Before wallet signing, show a mandatory preview modal.

Preview modal must include:
- Number of accounts being closed
- Recoverable SOL
- Estimated network fees
- Net SOL expected
- Number of transactions required
- Warning that closed token accounts cannot be recovered
- Confirmation button
- Cancel button

No transaction should be sent until the user confirms this modal.

====================================================
4. CLOSING PROGRESS
====================================================

When closing multiple batches:
- Show progress clearly.
- Example:
  Closing accounts 1–10 of 27
  Transaction 1 of 3
- After each successful batch, update progress.
- If a batch fails, stop and show clear error.
- After completion, rescan wallet or update list so closed accounts disappear.

====================================================
5. SAFETY RULES
====================================================

Preserve all safety rules:
- Never close accounts with non-zero token balance.
- Never auto-select accounts.
- Skip NFT candidates.
- Skip uncertain accounts.
- Rent returns to connected wallet.
- No platform fee in MVP.
- No backend routes.
- No database changes.
