BLACKPEBBLE ADMIN + PLATFORM POLISH BEFORE COMMUNITY BOOST POOL

Before building Community Boost Pool, improve the existing platform management, market reliability, and user-retention systems.

Reason:
Community Boost Pool involves real community coordination and eventually real SOL. Before adding that, BlackPebble needs better admin visibility, cleaner market data, and better paper trading reset/season controls.

Do NOT build Community Boost Pool yet.

====================================================

1. ADMIN DASHBOARD
    ====================================================

Create a protected admin dashboard.

Route:
/admin

Access:
Only allow access for approved admin X accounts.

Use X user ID, not display name.

Add environment variable:

ADMIN_X_USER_IDS

If current signed-in X user ID is not in ADMIN_X_USER_IDS:

* show unauthorized
* do not expose admin data

Admin dashboard should include:

Platform Stats:

* total users
* total wallets/accounts
* total paper trades
* total paper volume
* active positions
* active orders
* total leaderboard users
* daily active users if available

Paper Trading Controls:

* reset one user to 100 SOL
* reset all users to 100 SOL
* clear open positions
* clear pending orders
* clear trade history
* reset leaderboard stats
* preserve user accounts
* preserve wallet links
* preserve X links
* preserve watchlists unless explicitly selected

Order Management:

* view all active orders
* filter by token
* filter by user
* cancel stuck pending orders

Markets:

* view market cache age
* force refresh market cache
* see last updated time
* see API failures if available

System Health:

* API status
* DB status
* market data status
* recent errors if available

Feature Flags:
Add simple feature flags if easy:

* enable/disable buy limits
* enable/disable TP/SL orders
* enable/disable multi-target TPs
* enable/disable experimental utilities

Do not overbuild. Admin dashboard can be simple and functional.

====================================================
2. PAPER TRADING RESET / SEASON SYSTEM

Problem:
If a user blows up their 100 SOL paper account, they need a way to continue practicing without abusing leaderboards.

Add a basic paper trading season/reset system.

User-facing behavior:

If user equity or cash balance falls below a small threshold, such as 1 SOL:

Show:

Paper Account Depleted

Your paper trading account is below 1 SOL.

You can reset to a fresh 100 SOL balance and start a new season.

Button:
Start New Season

When user starts new season:

* reset paper cash balance to 100 SOL
* close/clear open positions
* cancel pending orders
* archive old season stats if practical
* reset current season P&L
* reset current season trade history
* preserve account identity
* preserve wallet/X links
* preserve watchlist

Do not automatically reset without user confirmation.

Admin should also be able to reset users from admin dashboard.

Future-ready:
Design so leaderboard can later support seasons.

If full season archive is too much for this pass:

* implement safe reset now
* add clear TODO / schema plan for full season history

====================================================
3. MARKET DATA FRESHNESS IMPROVEMENT

Problem:
Markets page can show stale/rugged tokens for too long.

Improve market data reliability.

Requirements:

Markets page:

* show Last Updated timestamp
* add manual Refresh button
* shorten cache duration for Markets page if currently too long
* avoid showing obviously dead/rugged tokens when possible

Filtering rules:

* filter pairs with extremely low liquidity
* filter pairs with missing price/market cap
* filter pairs with no meaningful recent volume
* if token liquidity has collapsed or price action shows extreme rug behavior, deprioritize or remove from trending if practical

Do not over-filter legitimate volatile memecoins.

Data source:
Use existing DexScreener / current market data integrations where possible.

Avoid expensive polling.
Do not add always-on background workers.
Refresh only when page is open or user clicks refresh.
