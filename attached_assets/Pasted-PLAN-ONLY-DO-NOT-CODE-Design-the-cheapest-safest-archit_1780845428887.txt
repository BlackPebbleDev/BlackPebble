PLAN ONLY — DO NOT CODE

Design the cheapest, safest architecture for BlackPebble paper trading advanced orders.

Goal:
Integrate Buy Limit, Sell Limit, Take Profit, and Stop Loss orders with the existing paper trading system and the Mini Trade Planner.

This should let users plan a trade, apply it to the paper trading form, and optionally attach planned order rules without requiring expensive always-on polling.

MAIN PRODUCT GOAL

The user should be able to:

1. View token chart
2. Open Mini Trade Planner
3. Enter target MC and stop MC
4. Enter investment amount
5. Click Apply To Trade
6. Paper buy manually
7. Save planned take profit / stop loss rules to the position
8. Have BlackPebble check those rules when the user returns, refreshes, or opens Trading / Portfolio

This is a paper trading simulation, not real trading.

====================================================
IMPORTANT COST CONSTRAINT

Avoid expensive always-on infrastructure.

Do NOT recommend:

* constant background polling for all users
* checking every watchlist token
* checking inactive users every minute
* push notification workers
* real-time order execution loops
* expensive always-on candle scanning

Preferred MVP:

* refresh-based order checks
* check only active user positions
* check only tokens with open planned orders
* use existing market data where available
* optionally use candle history later only when needed

ORDERS TO PLAN

Plan architecture for:

    * buy amount

====================================================
EXECUTION MODEL OPTIONS

Analyze and compare:

OPTION A — Current-price refresh only

* On Trading/Portfolio load, fetch current MC/price
* If order trigger is crossed, execute simulated fill
* Cheapest
* Less historically accurate

OPTION B — Candle-history check on return

* Store last checked timestamp
* On user return, fetch candle history since last check
* Check high/low to see if stop or target was touched
* More accurate
* Higher API cost

OPTION C — Hybrid MVP

* Start with current-price refresh
* Add candle-history verification only for tokens with open orders and only when last checked is old enough

Recommend the best MVP path.

====================================================
ORDER RULES

For long paper trades:

Take Profit / Sell Limit:

* Trigger when current MC/price >= target

Stop Loss:

* Trigger when current MC/price <= stop

Buy Limit:

* For dip buys: trigger when current MC/price <= entry target
* For breakout buys later: trigger when current MC/price >= breakout target
* For MVP, recommend whether to support only dip buy limits first

OCO:

* Do not build yet
* But plan how take profit and stop loss can later cancel each other

Partial sells:

* Allow percentage-based sells later
* Recommend MVP:
    * 100% stop loss
    * optional 25/50/100% take profit
