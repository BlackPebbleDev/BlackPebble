====================================================
1. OPEN POSITIONS MOBILE UX
====================================================

Current issue:
Open Positions and Positions table require horizontal scrolling on mobile. This feels cheap and unfinished.

Fix:
On mobile, replace wide tables with stacked responsive position cards.

Each open position card must show:

- Token symbol
- Token name if available
- Quantity owned
- Position value in SOL
- Cost basis in SOL
- Average entry price
- Current price
- Unrealized PnL in SOL
- Unrealized PnL %
- Optional: slippage-adjusted entry info if available

Example mobile card:

BEAR
Quantity: 7.64M
Value: 9.84 SOL
Cost: 10.00 SOL
Avg Entry: $0.00008631
Current: $0.00008254
PnL: -0.1556 SOL (-1.56%)

No horizontal scrolling on mobile.

Desktop may keep table layout if it looks good.

====================================================
2. POSITION DETAILS
====================================================

Make position cards/rows expandable or tappable.

Expanded details should show:

- Entry timestamp
- Quantity
- Average entry
- Current price
- Position value
- Cost basis
- Unrealized PnL
- Trade history for that token
- Slippage paid on entry if stored

Do not require sideways scrolling.

====================================================
4. TRADE COUNT LABEL CLARITY
====================================================

Current issue:
“Total Trades” is unclear.

Clarify stats by separating:

- Total Executions = all buy/sell actions
- Closed Trades = completed position exits / realized trades

If there is only room for one stat card, rename “Total Trades” to “Executions” for now.

Leaderboard rules should continue using closed trades where appropriate.

====================================================
5. BEST TRADE DISPLAY
====================================================

Current issue:
Best Trade shows 0.00 SOL when there are trades but no winning closed trade.

Fix display logic:

If there is a winning closed trade:
Show largest positive realized PnL.

Example:
Best Trade
+0.42 SOL

If there are closed trades but no winners:
Show:
No winning trades yet

If there are no closed trades:
Show:
No closed trades yet

Do not show misleading 0.00 SOL unless the value is actually exactly zero.

====================================================
6. CASH / PNL AUDIT
====================================================

Run a calculation audit for partial sells.

Verify:

- Buy 10 SOL
- Sell 25%
- Sell 50%
- Sell 75%
- Sell remaining 100%

Confirm:

- Remaining token quantity updates correctly
- Cash balance updates correctly
- Realized PnL equals sell proceeds minus cost basis sold
- Unrealized PnL only applies to open position
- Total PnL = realized PnL + unrealized PnL
- Equity = cash balance + open position value
- Portfolio values persist after refresh

If any mismatch is found, fix it.
