FEED TOKEN LINK + LIVE CALLOUT PERFORMANCE UPDATE

Current issue:

Token names in Feed cards are not clickable when users call a token, publish a thesis, or create related token activity.

Additionally, callout posts currently do not display the ongoing performance of the call.

BlackPebble is becoming a reputation platform.

Users should immediately see whether a call is winning or losing directly from the feed.

====================================================
PART 1 — CLICKABLE TOKEN LINKS

Expected behavior:

Any token name/symbol shown in the Feed should be clickable and navigate directly to that token’s BlackPebble token page.

Apply to feed event types:

* Call Published
* Thesis Published
* Trade Opened
* Trade Closed
* Leverage Opened
* Leverage Closed
* Liquidation
* Achievement tied to token
* Journal entry tied to token if shown publicly later

Requirements:

* Token name/symbol visually styled as clickable.
* Use BlackPebble V2 styling.
* Preserve existing View Token buttons if present.
* If token mint/address exists, route directly to token page.
* If only symbol exists, use existing token lookup logic.
* Mobile friendly.
* No layout shift.
* No broken links.

====================================================
PART 2 — LIVE CALLOUT PERFORMANCE

Upgrade call feed cards to display real-time performance.

Current feed example:

@username called PUFFY at $27K MC

New desired format:

📣 CALL

@username called PUFFY

Called:
$27K MC

Current:
$91K MC

Current:
3.37x

ATH:
5.82x

View Token

====================================================
PART 3 — DYNAMIC PERFORMANCE TRACKING

Current X and ATH X should NOT be static.

Calculate dynamically.

Use:

Current X =
Current Market Cap / Called Market Cap

ATH X =
ATH Market Cap / Called Market Cap

Requirements:

* Update automatically as token moves.
* Do not require user edits.
* Do not require re-publishing the call.
* Preserve original called MC forever.
* Preserve original call timestamp forever.

A call should become more impressive (or more embarrassing) over time.

Keep premium look.

* Author

* Thesis preview
* Timestamp
