Portfolio Page Structure

Current concern:

If a trader has many positions, a full analytics panel for every position creates excessive scrolling and poor usability.

Implement a 3-level information hierarchy.

Level 1 — Portfolio Overview (Default)

Each position card should remain compact.

Display:

* Token
* Unrealized P&L
* ROI %
* Entry MC
* Current MC
* MC Multiple (if available)

This should allow many positions to be viewed quickly.

⸻

Level 2 — Expanded Position

When expanding a position:

Display:

* Entry MC
* Current MC
* MC Multiple
* Position Value
* Cost Basis
* ROI
* Unrealized P&L
* Hold Time
* Quantity

This is the trader-focused view.

⸻

Level 3 — Full Position Detail

Add a dedicated Position Detail page/modal/screen.

This is where the full analytics live.

Include:

Market Cap Analytics

* Entry MC
* Current MC
* Peak MC
* MC Multiple
* MC Gain
* Drawdown From Peak

Position Analytics

* Avg Entry MC (prefer MC language over token-price language where possible)
* Position Value
* Cost Basis
* Quantity
* Unrealized P&L
* ROI
* Hold Time
* Executions
* Avg Slippage

Trade History

* All executions
* Buy history
* Sell history
* Timestamps
* Slippage per execution

This is the “deep analytics” screen.
