Add Guest Paper Trading Mode without breaking signed-in persistence.

Do not add unrelated features.

Goal:
Let users try BlackPebble immediately without connecting wallet or X, but require sign-in to save trades permanently and qualify for leaderboard.

====================================================
1. GUEST MODE
====================================================

Allow users to:
- Search tokens
- View charts
- Use buy/sell paper trading
- Create temporary guest positions
- View temporary guest portfolio
- Use temporary guest history

No wallet or X required.

Guest trades must persist while using the site:
- survive tab switching
- survive navigation between pages
- survive page refresh if possible using localStorage/sessionStorage

Guest trades do NOT need to persist across:
- different devices
- cleared browser storage
- long-term app/database storage

Display clear label:
“Guest Mode”

Display warning:
“Guest trades are temporary. Sign in to save your portfolio and join leaderboards.”

====================================================
2. SIGN-IN / SAVE RULES
====================================================

Permanent saved trading requires sign-in.

Signed-in means:
- wallet connected now
- later X login when available

When signed in:
- trades save to PostgreSQL
- portfolio persists
- watchlist persists
- history persists
- leaderboard eligibility starts

Guest users should NOT appear on leaderboard.

====================================================
3. CONVERSION PROMPTS
====================================================

After a guest trade executes, show a small non-blocking prompt:

“Want to save this portfolio? Connect wallet to save trades permanently.”

Buttons:
- Connect Wallet
- Keep Testing

Do not block guest trading.

====================================================
4. OPTIONAL MIGRATION
====================================================

If a guest connects wallet after making guest trades, offer:

“Save guest portfolio to this wallet?”

Options:
- Save Portfolio
- Start Fresh

If Save Portfolio is selected:
- migrate guest positions/history/watchlist into the signed-in account
- avoid duplicates
- clear guest localStorage after successful save

====================================================
5. UI REQUIREMENTS
====================================================

Header should clearly show one of:
- Guest Mode
- Wallet connected
- X connected later

Trading page should work in guest mode exactly like signed-in mode, except persistence/leaderboard restrictions.

Portfolio page should show guest temporary stats when in guest mode.

Leaderboard page should show:
“Connect wallet or X to appear on the leaderboard.”

====================================================
6. SAFETY / DATA RULES
====================================================

Do not mix guest trades with signed-in trades unless user explicitly chooses Save Portfolio.

Do not overwrite signed-in portfolio with guest data.

Do not reset PostgreSQL data.

Do not break current wallet-based trading.

====================================================
7. QA
====================================================

Verify:
- Guest can buy without wallet
- Guest can sell without wallet
- Guest portfolio updates
- Guest history survives tab navigation
- Guest refresh behavior works as intended
- Guest does not appear on leaderboard
- Wallet user trading still saves to PostgreSQL
- Existing signed-in positions/history remain intact
- Save guest portfolio flow works or is clearly deferred
- App builds
- Typecheck passes
