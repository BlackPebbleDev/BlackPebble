Only modify Wallet Cleaner files.

Goal:
Make the Wallet Cleaner flow feel clearer, more complete, and more professional while keeping it fully isolated from the paper trading platform.

1. Hero section

Add/improve the hero section.

Recover trapped SOL from empty token accounts

Keep this professional, not hypey.

2. Wallet status card

After wallet connect, show a clear wallet status card.

Display:

* Connected wallet address, shortened
* Total wallet SOL balance
* Empty accounts found
* Recoverable SOL
* Estimated network fees
* Estimated net receive

Important:

* Total wallet SOL balance must be fetched from the connected wallet using live on-chain data.
* Use the existing Solana connection / Helius RPC.
* Do not hardcode wallet balance.
* Refresh wallet balance after scan and after successful close.
* If wallet balance fetch fails, show “Unavailable” without breaking the scan.

4. Empty wallet / clean wallet state

If the wallet has no closeable empty token accounts, show a successful clean state.

Display:
Wallet clean

Description:
No empty token accounts were found for this wallet.

Also show:

* Total wallet SOL balance
* Empty accounts found: 0
* Recoverable SOL: 0
* Estimated net receive: 0
* Scan Again button

Important:

* Do NOT show “Could not read token accounts from the network” for a valid empty wallet.
* Distinguish between a true RPC/network error and a successful scan with zero results.
* If getParsedTokenAccountsByOwner returns an empty array, treat it as success.

Wallet cleaned

After completion:

* Refresh wallet balance

8. Error handling

Fix scan and close errors cleanly.

Rules:

* Fresh wallet with no token accounts = clean wallet state, not error.
* Valid scan with zero closeable accounts = clean wallet state, not error.
* Real RPC/network failure = show network error.
* Partial close failure = show what succeeded and allow retry for remaining accounts.
* Do not duplicate error banners.
* Do not show “wallet clean” during real error states.

9. Safety rules

Preserve all safety rules:

* Never close accounts with non-zero token balances.
* Never auto-select accounts.
* Skip NFT candidates.
* Skip uncertain accounts.
* Rent returns to connected wallet.
* No platform fee in MVP.
* No backend routes.
* No database changes.
* Full preview required before signing.
